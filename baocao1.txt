Milestone 1: Initial Software Solution Design Report 
Section 1: User Research Findings and Key User Groups 
This section details the findings from the initial user research conducted to understand the needs and context for the proposed software solution. 
User Research Findings 
The initial research process included analyzing the WHO’s annual reports on global vaccination and comparing existing heal data dashboards. The key information is summarized below: 
  
● Finding 1 (Regarding economic burden): 
-	Evidence: A UNICEF report shows vaccine inequality, where children in the poorest households or low-income countries have much lower vaccination rates compared to other groups. 
-	Evidence Link: https://www.unicef.org/reports/state-worlds-children-2023 - 	Main point to cite: Emphasizes the “Immuniztion Inequity” section. 
● Finding 2 (Regarding data barriers): 
-	Evidence: The WHO admits that even though they have a huge amount of data, managing and accessing health data from paper or PDF reports at the local level is still a big challenge, causing delays in analysis. 
-	Evidence link: https://www.who.int/publications/i/item/9789240020924 
-	Main point to cite: See the “Challenges” section about the lack of connection and difficulties in gathering data from unstructured sources (like text reports). 
● Finding 3 (Regarding visual clarity): 
-	Evidence: Data from Our World in Data shows the need to use “World Average” lines as a benchmark to compare the situation of specific countries, helping to identify countries that are falling behind. 
-	Evidence Link: https://ourworldindata.org/vaccination 
-	Main point to cite: This page uses charts comparing vaccine coverage with GDP, showing the value of comparing a country to the overall average (Baseline).

Section 3: User Interface Designs 
This section presents the initial user interface (UI) designs for the key web pages of the software solution, progressing from initial sketches to digital wireframes. 
3.1 Webpage 1: Overview Dashboard (Level 1) 
Wireframe 
The wireframe shows the structural layout and information architecture, without final styling. 
  
  
Explanation 
This page provides a high-level summary of global infection and vaccination trends. It serves as the entry point for users before moving into deeper analysis: 
  
●	Layout: A top section displays KPI summary cards, followed by a global trend chart. The sidebar navigation remains fixed for consistency. 
●	Key Elements: 
-	KPI cards showing global indicators. 
-	Global trend chart. 
-	Quick comparison preview. 
● User Flow: Users first review global statistics → observe trend patterns → navigate to Economic Perspective or Above Average Analysis for detailed insights. 
3.2 Webpage 2: Mission Statement Page (Level 1) 
Wireframe 
  
 
 
  
Explanation 
This page is designed to introduce the purpose and mission of the project, helping new users understand the dashboard’s goals and get to know the stakeholders and the team: 
  
●	Layout: Uses a two-main-column layout: A fixed navigation bar (sidebar) on the left (240px) and a scrollable main content area on the right. In the content area, the parts are arranged vertically, including: the main banner, persona cards in a 3-column grid, and a team information table. 
●	Key Elements: 
-	Hero Banner: Uses a professional gradient color with a strong mission message and trust indicators (Data from WHO, updated annually). 
-	Persona Cards: Each card shows an AI portrait, the role, and the needs of a sample user. The shadow effect when hovering the mouse helps increase interactive feedback (Nielsen: Visibility of system status). 
-	Team Table: Lists the Name, Student ID, and Role in detail to identify project responsibilities. 
● User Flow: The user starts at the mission banner → Scrolls down to understand the web users → Views the project team information → Uses the left menu bar to switch to data analysis pages. 
 
3.3 Webpage 3: Global Comparison Page (Level 2) 
Wireframe 
  
Explanation 
This page enables cross-country comparison without economic grouping, allowing users to identify differences across nations: 
  
●	Layout: A filter bar at the top allows selection of infection type and year. Below it, a sortable comparison table presents country-level data. 
●	Key Elements: 
-	Filter controls. 
-	Sortable comparison table. 
-	Visual indicators for high/low values. 
● User Flow: Users apply filters → compare country data → sort to identify patterns → move to 3.4 or 3.6 for deeper analysis.  
3.4 Webpage 4: Economic Perspective – Shallow Glance (Level 2) 
Wireframe 
  
Explanation 
This page is designed to let users explore infection data based on economic status, providing a comparative view of the disease burden Developed, Developing and Emerging economy countries: 
  
●	Layout: The content structure is stacked in order: Filter Bar → Detailed Data Table → Economic Summary Table. This top-down arrangement hleps optimize the data exploration process: filter data → check details → view summary. 
●	Key Elements: 
-	Filter Bar: Includes 3 drop-down menus (Economic Status, Infection Type, Year) with “Apply” and “Reset” buttons. This follows the principle of (“User control and freedom”. 
-	Main Data Table: Displays country names, infection rates and column sorting buttons to support quick searches. 
-	Economic Summary Table: Groups data by economic phase to compare total cases and average rates between the groups. This is a key feature that helps answer questions about economic-health differences. 
● User Flow: The user selects filter criteria → Clicks Apply → The detail table updates the corresponding country data → The user scrolls down to compare overall between economic groups. 
3.5 Webpage 5: Detailed Country View & Export (Level 2) 
Wireframe 
  
Explanation 
This page provides detailed country-level data and supports data export for external use: 
  
●	Layout: Users select a country at the top. Below, detailed statistics are displayed in chart and table format, with an export option clearly visible. 
●	Key Elements: 
-	Country selector. 
-	Historical trend chart. 
-	Data table. 
-	CVS export button. 
● User Flow: Users select a country → review detailed trends → export structured data if needed. 
3.6 Webpage 6: Above Average Analysis – Deep-Dive (Level 3) 
Wireframe 
  
Explanation 
This page is designed to highlight countries with infection rates exceeding the global average, helping health authorities quickly identify high-risk areas needing priority intervention: 
  
●	Layout: Keeps the consistent sidebar structure. The main content area focuses on Key Performance Indicator (KPI) cards at the top before showing the large data table below to create visual priority. 
●	Key Elements:  
-	Global Average Indicator Card: Designed to stand out the most with brand colors, acting as a benchmark for the whole website. 
-	Warning Banner: A red notification strip clearly shows the filtering rule (Ex: “Only showing countries above 35.1” 
● User Flow: The user selects Year and Infection Type → The KPI cards update to the new world average number → The user sees the confirmation banner → Scans the data table and relies on badge colors to identify the countries nedding the most attention.
